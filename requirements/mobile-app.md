# Mobile Reading App Requirements

## Status
The mobile reading app is a v2 requirement and is not part of the v1 frontend demo.

## Goal
Provide a mobile app for manual meter-reading workflows in municipal sites, especially when readings happen in remote buildings or locations with unreliable connectivity.

## Required V2 Capabilities
- App-based login.
- Offline local buffering of readings when no internet connection is available.
- Automatic synchronization when internet connection becomes available.
- Reading workflow support for organized meter-reading rounds.
- Camera support.
- QR-code support for identifying meters/datapoints.
- Clear sync status for pending, synced, and failed readings.
- Secure handling of locally buffered readings.

## Relationship To V1
V1 supports CSV import and API ingestion only. The mobile app should be planned, but not implemented for the first frontend demo.
